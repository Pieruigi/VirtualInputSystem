Features

The MIS provides you with the most common input controls for your mobile game:

  1. Virtual Joystick

  2. Virtual Button

  3. Tap

  4. Swipe.



In the following section you will be explained how to use each of them.



Virtual Joystick

The virtual joystick alown you to simply simulate gamepad analogic axis.

Open the JoystickAndButtonScene to see an example.

Lets see how to set up step by step:

  1. Create a new scene or open an existing one

  2. Add a new canvas ( you can use also an existing one )

  3. Go to the Assets/VirtualInputSystem/Prefabs/InputUI folder

  4. Drag and drop the VirtualJoystick prefab into the canvas



Now check the Joystick script in the prefab inspector, you will see the following parameters:

  • HorizontalAxis: the action you want to map to the horizontal axis ( for example move or look left/right, or whaterver else you want to do )

  • VerticalAxis: the same as the horizontal axis ( for example move forth and back )

  • Stick: the analogic stick ( the child object )

  • MoveRange: the maximum distance you want the stick to be moved ( normally it will be set according to the width and height in the RectTransform when you resize the joystick )

  • ResetTime: how long it will take the stick to get back in the rest position; if you set zero the stick will snap in one single frame

If you want for example to add a Joystick to move your player name the axis “Horizontal” and “Vertical” and in your PlayerController script add the following code:



Vector2 moveInput = new Vector2(VirtualInput.GetAxis(“Horizontal”), VirtualInput.GetAxis(“Vertical”));

MoveCharacter(moveInput); // Some method to move the character around



This should reminds you the Unity InputSystem:

VirtualInput.GetAxis(“AxisName”) returns a float value between -1 and 1 depending on the stick position.

There is also a GetAxisRaw(“AxisName”), which returns 0 if the stick rest in place, otherwise returns 1 or -1 depending whether the axis value is positive or negative.

You can add as many virtual joysticks as you want: for example you want your character to move and look around you can add two joysticks, one to the left and the other to the right. Then call the left joystick axis “Horizontal” and “Vertical” while the right joystick axis “LookHorizontal” and “LookVertical”.

Now in your PlayerController script you can call both VirtualInput.GetAxis(“Horizontal”) and VirtualInput.GetAxis(“LookHorizontal”) for example in order to move and look horizontal.



Virtual Button

You can add many virtual buttons in your mobile game.

Check the VirtualJoystickAndButton example in which the player can jump and glide.

Lets step by step how to put a button into a new scene:

  1. Create a new scene or open an existing one

  2. Add a new canvas ( you can use also an existing one )

  3. Go to the Assets/VirtualInputSystem/Prefabs/InputUI folder

  4. Drag and drop the VirtualButton prefab into the canvas

In the inspector you can see the Button script which only has one parameter to map the action.

For example call the button “Jump” and in your PlayerController script add the following code:

if(VirtualInput.GetButtonDown(“Jump”))

{

PlayerJump(); // Some method to let the player jump

}

if(VirtualInput.GetButton(“Jump”))

{

PlayerGlide(); // You know, you may want your player to glide and not to fall down

}

if(VirtualInput.GetButtonUp(“Jump”)){

{

FallAndDie(); // Ok maybe you want to do something else here

}



As for the normal input system you can call VirtualInput.GetButton(“ButtonName”) which returns true if the player holds the button down, othwerwise false

VirtualInput.GetButtonDown(“ButtonName”) returns true as soon as the button is pressed and

VirtualInput.GetButtonUp(“ButtonName”) returns true as soon as the button is released ( otherwise they both return false ).

You can add as many buttons as you want, for example to “Jump” or “Shoot” or “Interact”.



Tap and swipe

Open the TapAndSwipeScene to see an example.

Lets see step by step:

  1. Create a new scene or open an existing one

  2. Add a new canvas ( you can use also an existing one )

  3. Go to the Assets/VirtualInputSystem/Prefabs/InputUI folder

  4. Drag and drop the TaperSwiper prefab into the canvas

Check the inspector, you can see two scripts on the prefab: Taper and Swiper.

They can be enabled or disabled: for example if you just want to tap you can disable the Swiper script ( or even delete it ).

Taper has only one parameter, TapTime, which defines for how long you can hold down your finger on the screen before the tap fail: for example by setting TapTime=1 tap fails if you hold your finger for more than one second.

In your PlayerController call VirtualInput.GetTap(out TapData) and VirtualInput.GetSwipe(out SwipeData) to check for tap and swipe.

VirtualInput.GetTap(out TapData) returns true when player taps the screen, otherwise false; if true you can read the position and the duration of the tap in the TapData structure.

VirtualInput.GetSwipe(out SwipeData) returns true when player swipes the screen, otherwise false; if true you can the position and the delta ( position – delta = previsousFramePosition ) in the SwipeData structure.
